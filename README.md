# OTP Service - Сервис аутентификации пользователей через Email, SMS и Telegram

## Архитектура и стек проекта
Проект реализован с использованием следующих технологий:
- Java 21
- Spring Boot 4.0.6
- Postgres 17
- Liquibase
- Docker
- JWT
- SMPP Client
- Java Mail Sender
- Telegram Bot API

Сервис обладает следующим функционалом:
- Аутентификация и авторизация пользователей с ролевой моделью через JWT (`USER`, `ADMIN`)
- Интеграция с 3 каналами сообщения: Email, SMS (эмуляция) и Telegram Bot
- Безопасная генерация уникальных 6-значных временных кодов с механизмом валидации и устаревания (`ACTIVE`, `EXPIRED`, `USED`)
- Админская панель для обновления OTP конфигурации и работы с пользователями

## Quick Start
### 1. Клонирование проекта
```git clone https://github.com/yofujitsu/otp-project.git```

```cd otp-project```

### 2. Создание .env файла
```
TG_TOKEN=your-token
JWT_SECRET=your-secret
EMAIL_USERNAME=your-email
EMAIL_PASSWORD=your-email-password
```
Чтобы данный файл был виден IDE можно установить плагин `EnvFile` и выбрать путь к созданному файлу в настройках конфигурации.

#### Работа с Telegram
Для интеграции с Telegram необходимо создать собственного бота в @BotFather и ввести токен доступа в `.env` файл.
#### Работа с Email
Для работы с почтой используется SMTP сервер Яндекса, поэтому надо использовать почту `@yandex.ru`. Обычный пароль от почты не подойдет, в сервисе Яндекс ID необходимо создать `Пароль приложения` и вставить его в `.env` файл.
Ваша почта Яндекс используется только для отправки сообщений, для приёма сообщений можно использовать почту в любом другом домене.
#### Работа с SMS
Для работы по SMS необходимо установить эмулятор SMPPSim и запустить его локально. В поле номера телефона при регистрации можно вводить любой номер, так как отправка сообщения будет эмулироваться. Результат успешной отправки сообщений можно отслеживать в консольных логах SMPP эмулятора.

### 3. Запуск базы данных
docker-compose up -d db

### 4. Сборка проекта
mvn clean package

### 5. Запуск приложения
mvn spring-boot:run
